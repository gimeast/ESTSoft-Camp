Redux란 무엇인가?

Redux는 JavaScript 애플리케이션의 예측 가능한 상태 컨테이너입니다.

🎯 목적

복잡한 애플리케이션 상태를 체계적으로 관리

📱 사용처

React, Vue, Angular 등 모든 JS 프레임워크

🔄 방식

단방향 데이터 플로우로 상태 변경

🛡️ 안정성

예측 가능하고 디버깅이 쉬운 상태 관리

Redux의 핵심 개념

🏪 Store

애플리케이션의 전체 상태를 보관하는 단일 저장소

⚡ Action

상태 변경을 설명하는 평범한 JavaScript 객체

🔄 Reducer

현재 상태와 액션을 받아 새로운 상태를 반환하는 순수 함수

📤 Dispatch

액션을 스토어에 전달하여 상태 변경을 트리거하는 함수

Redux의 3가지 원칙

1️⃣ 단일 진실 공급원 (Single Source of Truth)

애플리케이션의 모든 상태는 하나의 스토어 내 객체 트리에 저장

2️⃣ 상태는 읽기 전용 (State is Read-Only)

상태를 변경하는 유일한 방법은 액션을 발생시키는 것

3️⃣ 순수 함수로 변경 (Changes are Made with Pure Functions)

상태 변경은 순수 함수인 리듀서로만 수행

단방향 데이터 플로우의 장점

  • 예측 가능한 상태 변경
  • 디버깅의 용이성
  • 시간 여행 디버깅 지원
  • 상태 변경 추적 가능

언제 Redux를 사용하는가?

🤝 상태 공유

여러 컴포넌트가 같은 상태를 공유할 때

📈 복잡한 상태

상태 변경 로직이 복잡하고 예측하기 어려울 때

🔍 디버깅 필요

상태 변경 과정을 추적하고 디버깅해야 할 때

🏗️ 대규모 앱

중간 규모 이상의 애플리케이션 개발 시

💡 주의: 간단한 애플리케이션에서는 오히려 복잡성을 증가시킬 수 있습니다.

실제 코드 예시

Action 정의

const INCREMENT = 'INCREMENT' const DECREMENT = 'DECREMENT' const increment = (amount) => ({ type: INCREMENT, payload: amount }) const decrement = () => ({ type: DECREMENT })

Reducer 작성

function counterReducer(state = { count: 0 }, action) { switch (action.type) { case INCREMENT: return { count: state.count + action.payload } case DECREMENT: return { count: state.count - 1 } default: return state } }

Store 생성 및 사용

Store 생성

import { createStore } from 'redux' const store = createStore(counterReducer)

상태 사용

// 현재 상태 조회 console.log(store.getState()) // { count: 0 } // 상태 변경 store.dispatch(increment(5)) console.log(store.getState()) // { count: 5 } store.dispatch(decrement()) console.log(store.getState()) // { count: 4 } // 상태 변경 구독 store.subscribe(() => { console.log('상태가 변경되었습니다:', store.getState()) })

Redux의 장단점

✅ 장점

  • 예측 가능한 상태 관리
  • 중앙 집중식 상태 저장
  • 시간 여행 디버깅
  • 핫 리로딩 지원
  • 미들웨어 생태계
  • 테스트 용이성

❌ 단점

  • 학습 곡선
  • 보일러플레이트 코드
  • 작은 앱에는 과도함
  • 불변성 유지 필요
  • 추가적인 의존성

💡 팁: Redux Toolkit(RTK)을 사용하면 많은 단점을 해결할 수 있습니다!
